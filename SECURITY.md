# Security Policy

## Supported Versions

The `main` branch is the supported development line until the first production
release is tagged.

## Reporting a Vulnerability

Report suspected vulnerabilities privately to the project owner or client
security contact. Do not open public issues for security vulnerabilities.

Include:

- Affected route, feature, or integration.
- Steps to reproduce.
- Potential impact.
- Any logs or request IDs that help investigation.

## Handling

Security fixes should be prioritized, reviewed, and released separately from
unrelated feature work when practical.
